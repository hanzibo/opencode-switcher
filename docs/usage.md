# OpenCode Switcher 使用指南

**OpenCode Switcher** 是一个专为 Linux GTK3 桌面环境设计的系统托盘应用。它允许用户通过极简的搜索面板，在不同的 OpenCode (CLI) 会话之间快速搜索、管理和切换，并内置了高精度的剪切板历史管理、自定义模板提示词和系统集成功能。

---

## 一、 系统快捷键与面板控制

### 1. 唤起与隐藏
- **全局唤起/隐藏**：`Ctrl+Shift+Space`
  - **X11**：通过 `pynput` 全局后台线程直接监听。
  - **Wayland**：通过专属 GNOME Shell 扩展监听系统快捷键，并发送信号至 Unix 套接字进行唤起。
- **隐藏面板**：
  - 按下 `Escape` 键。
  - 点击面板外部（失去焦点时自动隐藏，但在弹窗编辑状态下会自动锁定防止误触隐藏）。

### 2. 标签页切换
- **切换到【会话管理】**：`Ctrl + 1`
- **切换到【剪贴板历史】**：`Ctrl + 2`

## 二、 会话管理面板 (Session Switcher)

会话管理面板（位于第一个标签页）可以实时从 OpenCode 本地数据库（`~/.local/share/opencode/opencode.db`）读取历史会话。

### 1. 列表操作与状态识别
- **活动状态检测**：应用会自动读取 `/proc` 进程信息或使用 `pgrep` 探知正在运行的会话，并在列表中以绿色指示灯 🟢 **Live** 标出。
- **删除会话**：选中某一历史会话后，按下 `Delete` 键可直接将其从数据库中删除。
- **重命名会话**：选中某一历史会话后，按下 `Ctrl + R` 可弹出重命名窗口，快速修改会话标题。
- **导航与打开**：
  - 使用 `Up` / `Down` 键选择会话，`Enter` 键确认。
  - 应用会自动探知系统内安装的终端（按优先级：`Ptyxis` → `GNOME Terminal` → `Console/kgx` → `Black Box`），并在对应的终端里拉起该 OpenCode 会话。

### 2. 快捷指令 (Virtual Commands)
在搜索框中输入以下前缀指令，可以触发高级操作。支持通过 `Tab` 键自动补全指令前缀：

| 指令 | 作用描述 | 示例与使用细节 |
| :--- | :--- | :--- |
| `/new` | 在当前目录下开启一个新的 OpenCode 会话 | 输入 `/new` 后按回车 |
| `/open` | 弹出目录选择弹窗，选择并开启一个新会话 | 输入 `/open` 后按回车 |
| `/gm <query>` | 快捷向 Gemini 提问 | 输入 `/gm 帮我写一段 python 冒泡排序`。系统会自动复制内容，打开默认浏览器并聚焦到 Gemini Web App，并在后台自动模拟输入（Ctrl+V + 回车）发送给 AI。 |
| `/google <query>` | 快捷发起 Google AI 搜索 | 输入 `/google Rust 异步编程最佳实践`。系统会自动复制内容，打开浏览器进行搜索并自动提交。 |

## 三、 剪贴板历史与提示词模版 (Clipboard & Prompts)

剪贴板管理面板（位于第二个标签页）提供强大的数据收集与 AI 工作流联动能力。

### 1. 高精度内容识别与分类
- **智能分类**：应用利用启发式算法自动将您的复制历史分类为：
  - 📋 **文本** (`Text`)
  - 🌐 **链接** (`Link`)
  - 💻 **代码** (`Code`)
  - 🖼️ **图片** (`Image`)
- **语言探测**：对于被识别为代码段的项，会自动探测出具体编程语言（如 `PYTHON`、`JAVASCRIPT`、`SHELL`、`C++` 等），并在 UI 列表中直观展现大写语言标签。
- **图片预览**：点击图片历史项可以直观预览图片内容（图片历史数据保存在系统缓存 `images/<hash>.png` 下）。

### 2. 拖拽 (Drag & Drop) 与粘贴支持
- 支持直接通过**鼠标拖拽（DND）**将剪贴板历史列表中的某一项（包含文本、代码、图片文件）拖动并放入其他软件中（如 VS Code、浏览器输入框或文件管理器）直接实现粘贴或上传。
- 支持按 `Enter` 键重新将历史项写回系统剪切板。

### 3. 自定义分类文件夹 (Categories)
- 在左侧侧边栏中可以创建不同的自定义分类文件夹。
- 支持右键点击历史记录，选择 **移动到分类...** 或 **新建分类...**，将常用内容持久化保存，避免随历史滚动被冲刷掉（历史记录采用 FIFO 150 限制）。

### 4. 模版化提示词工程 (Prompts Engineering)
您可以在剪贴板分类中为常用的 AI 工作流配置模板提示词（Prompts）：
- **占位符 `${&}`**：在定义 Prompts 时，可以使用 `${&}` 占位符代表“当前剪贴板中复制的内容”。
  - 示例：`请对以下代码进行安全审计：\n${&}`。
  - 当您复制了一段代码并点击这个 Prompt 按钮时，系统会自动将代码替换到 `${&}` 位置，将组装好的完整提示词写回剪切板，方便您一键发送给 AI 助手。
- **转义符 `\${&}`**：如果您需要在最终输出中保留字面量 `${&}`，请使用反斜杠转义（`\${&}` 将输出为字符 `${&}`）。
- **便捷操作**：在 Prompts 模板配置界面中，提供一键插入 `${&}` 占位符的便捷按钮。

### 5. 多参数模板与 “Dynamic Copy”（动态复制）
对于更复杂的 AI 提问场景，应用支持高级的多参数模板，其占位符语法为 `${序号[:提示词][=默认值]}`：
- **占位符格式**： `${1:变量名称=默认文本}` 或 `${2:API Key}`。
  - **序号**：用于区分和引用不同的参数输入框。
  - **提示词**（可选）：用作参数输入框旁的标签说明。
  - **默认值**（可选）：用作输入框的初始预填值。
- **Dynamic Copy 操作**：
  - 当一个提示词模板包含此类参数占位符时，列表行上会直观地显示绿色的 **Dynamic Copy** 标记。
  - **右键**该项并选择 **Dynamic Copy**，将弹出一个专门的参数填充对话框。
  - **实时双栏预览**：对话框左侧为参数输入表单（根据占位符定义自动生成标签和默认值），右侧展示实时渲染后的最终提示词效果。
  - 点击底部的确认按钮，即可直接将最终渲染的文本拷贝到剪切板，完成快速定制输入。

### 6. 配置备份与还原 (Backup & Restore)
为防止由于系统重装、迁移或误删导致的数据丢失，应用在配置中心提供了数据备份和还原功能：
- **数据备份 (Backup)**：
  - 点击提示词配置界面中的 **Backup** 按钮，选择目标保存目录。
  - 系统会将包含所有自定义分类、提示词模板、剪贴板历史记录及图片缓存的配置目录 `~/.config/opencode-switcher` 统一打包为一个标准的压缩文件：`opencode-switcher-backup-YYYYMMDD_HHMMSS.tar.gz`。
- **数据还原 (Restore)**：
  - 点击 **Restore** 按钮，选择此前备份的 `.tar.gz` 文件。
  - 系统将解压并完整覆盖当前的配置数据，自动刷新面板列表，实现无缝数据迁移。

## 四、 服务管理与日常维护

应用安装脚本会部署 Systemd 用户服务，您可以通过系统标准命令管理后台服务：

- **查看服务状态**：
  ```bash
  systemctl --user status opencode-switcher.service
  ```
- **启动/停止服务**：
  ```bash
  systemctl --user start opencode-switcher.service
  systemctl --user stop opencode-switcher.service
  ```
- **重启服务**：
  ```bash
  systemctl --user restart opencode-switcher.service
  ```
- **查看运行日志**：
  在生产环境下，应用运行日志会自动输出到 `run.log` 文件中，默认拥有 10MB 的轮转回滚保护。
