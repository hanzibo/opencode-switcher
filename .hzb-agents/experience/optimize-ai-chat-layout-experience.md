# 优化 AI 看盘界面布局 开发经验总结

> **分支名**：`optimize-ai-chat-layout`  
> **开发周期**：2026-07-04 至 2026-07-04  
> **关键词**：`AI看盘` `界面布局` `复制按钮` `无用代码清理` `布局优化`

## 一、经验与教训总结

### 1.1 做得好的地方
- **UI 精简化与痛点导向**：移除用户不常用、非核心的全局复制功能，在视觉上实现了界面“瘦身”，提高了操作栏的简洁性与核心操作（对话选择与关闭）的触达率。
- **干净的死代码清理**：在移除 UI 控件的同时，完整地识别并消除了其在 Python 后台的本地 clicked 事件绑定、布局装载代码，做到了零逻辑残留。

### 1.2 需要改进的地方
- **服务重新部署的感知**：通过 `./install.sh install` 重新编译部署修改文件时，如果系统服务原本处于活跃运行状态（Running），仅仅 enable 服务并不会自动重启该进程加载最新代码，需要手动执行一次 `systemctl --user restart opencode-switcher` 强制重载。后续对于有常驻服务运行的系统，重装后必须记得重启服务。

## 二、关键问题与解决方案记录

### 问题1：移除冗余的全局复制功能，同时保留局部对话复制
- **问题描述**：需要将右上角用于复制整个会话文本的“📋 复制”按钮去掉，但不能影响已有的局部气泡复制。
- **原因分析**：大图预览与每条对话附带的局部复制（📋 复制回答、📋 复制输入）更加切合碎片化的复制使用场景。全局复制会将 Markdown 原始标记以及多余的分隔符混杂在一起，实际使用率极低。
- **解决过程**：
  1. 定位到 `ai_chat_panel.py` 中的 `_btn_copy_ai` 控件。
  2. 确定该控件没有被外部模块作为跨类 API 访问。
  3. 彻底删除控件定义、样式绑定、局部回调函数 `on_copy_ai_clicked` 及其所在的 Gtk.Box 装载操作。
- **最终方案**：彻底删除了该全局复制按钮，将顶部标题操作栏恢复为纯粹的“标题 + 历史列表 + 关闭”结构。
- **预防建议**：在产品迭代过程中，要根据实际使用反馈定期审计并清理无用或冗余的 UI 元素，保持界面的极简与高效。

## 三、技术要点沉淀

- **常驻服务的部署与重载模式**：
  在基于 systemd 用户服务管理的常驻 GUI 桌面应用开发中，执行文件被拷贝更新后，必须执行以下链式指令以确保最新代码生效：
  ```bash
  ./install.sh install && systemctl --user restart opencode-switcher
  ```
